Each agent should assess EACH category against: (a) behavioral science, (b) this user's cognitive profile, (c) the app's guardrails. 6A. SESSION PROTOCOLS (Operational Sequences) The idea: Habits that are multi-step sequences get a "session" format instead of a single checkbox. A laundry session has ordered steps. A grooming protocol has a morning stack and an evening stack. The user completes steps in sequence; the system tracks session completion, not individual step streaks. Questions: Does session-format add cognitive load or reduce it for this profile? What's the science on chunked behavioral sequences vs discrete tasks for INTJ/Analytical types? What's the risk model? 6B. MAINTENANCE CADENCE (Non-Binary Rhythms) The idea: Some behaviors aren't once-per-day — they're interval-based. Water filter replacement (monthly). HVAC filter (quarterly). Weekly deep clean. These don't fit the daily action stack. They need a different temporal logic: "due", "upcoming", "overdue" — with no shame framing, just state reporting. Questions: How should interval-based maintenance behaviors be surfaced without creating a to-do list aesthetic? What's the science on maintenance behavior adherence for this profile type? 6C. HYDRATION / RHYTHMIC BEHAVIORS The idea: Hydration isn't a checkbox — it's a frequency target. The user needs to re-establish hydration as a rhythm, not an event. This may require time-distributed gentle prompting, not a daily completion toggle. Same applies to: movement breaks, sunlight, posture checks, breathing, short mental resets. Questions: What's the science on interval-based health prompting vs daily completion for hydration specifically? What cadence logic works for Analytical/Competition profiles without triggering score-proxy behavior? How do we surface frequency without creating a counter? 6D. FINANCIAL VISIBILITY (Operational Clarity Layer) The idea: The user needs financial habits integrated — not as a spreadsheet, but as operational clarity. Runway visibility. Recurring obligations. Spending awareness. Restoration vs depletion framing. Eventually: a lightweight financial operating layer inside the app. Questions: What is the minimum viable financial visibility feature that provides operational clarity without becoming a budgeting app? What does behavioral science say about financial habit formation for high-Achiever, high-Analytical profiles? What are the psychological risks of financial metrics inside an achievement-oriented system? 6E.
